Designing a Database and Database Normalization

Designing a Normalization is the process of removing anomalies and redundancies from the database design. The normalization process requires certain set of rules or design to reduce or eliminate anomalies, it is known as normal forms.

Anomalies

  • Insertion Anomaly - this occurs when can't enter data because other data are missing.
  • Update Anomaly - occurs when data stored in more than place. If the data needs to be changed or updated, it would need to be update all over or else it will be inconsistent.
  • Deletion Anomaly - occurs where deleting a piece of data inadvertly causes other data to be lost.

First Normal Form

First normal form involves removing all repeated groups or arrays. An entity is in the first normal formals if every attribute represents one value, no repeating groups or arrays, and each row is unique. If a table is not in first normal form, you will notice that it will not have a single value store in a column.

Second Normal Form

Second Normal Forms removes functional dependencies. Functional dependencies refer to groups of related attributes that form subthemes within an entity.

Third Normal Form

Third normal form involes removing trasient dependencies. Transcient dependencies is when one attribute depends on another attribute for its meaning.

Note: There are other types of normal forms, but we will briefly define first, second, and third normal forms for now.

Physical Design

Physical design is the design adapted to the RDBMS and system constraints and features.

Additional Database Terminology

  • Constraints - limits on what the database will do.
  • Primary Key - a unique key that identifies each row in a table.
  • Foreign Key - a primary key that is repeated in another table to create a link between the tables.
  • Composite Key - a key that consists of more than one attribute.
  • Natural key - a natural key is a single column or set of columns that uniquely identifies a single record in a table, where the key columns are made up of real data
  • Surrogate Key - an artificial key often just auto-incremented numbers.

Denormalization

Denormalization is the process of joining tables that were seperated in the normalization process to improve performance.
